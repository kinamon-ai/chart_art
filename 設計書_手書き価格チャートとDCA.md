# 設計書：手書き価格チャート × ドルコスト平均シミュレータ

本文書は **実装より先に認識を合わせる**ためのものです。現状実装との差分、あるべき体験、データの流れ、未決事項を整理します。

---

## 1. 背景と問題の整理

### 1.1 元コンテンツ（企画・断片コード）での前提

`/ドルコスト計算シミュレーター.md` の断片には、処理のつながりとして次がある。

- **`calculatePriceVariations()`** — チャート側の入力・描画結果から「価格変動」の列を作る関数（本体は別ファイル／未復元）。
- **`calculateDCAResults(priceVariations)`** — 変動率配列から積立・評価額などを算出。
- **`calculateAndDisplayResults()`** — 変動リスト作成 → DCA → 結果グラフ → サマリー → 共有。

また `追加機能の詳細.md` でも **「購入価格はチャートの変動率に基づいて計算する」** と説明がある。

つまり **最大の売りは「ユーザーが価格の動きをチャート上で自分で決める（入力の主通路がチャート）」こと**であったと整理できる。

### 1.2 現状実装（Next.js アプリ）

- **入力**：期間ごとの **数値入力グリッド** と **変動率のランダム生成**。
- **算出**：変動率配列 → **`calculateDCAResults`**（`lib/dca.ts`）。ロジックは断片と同型。
- **結果**：Chart.js で **損益率バー表示**・サマリー・共有など。

つまり **DCA 以降や共有は近いが「チャート由来の変動」の体験が欠落**しており、元作品とは **コンセプト上致命的に異なる**。本設計書はそのギャップを埋めることをスコープの中心とする。

## 2. 目標となるユーザー体験（要件レベル）

### 2.1 コア体験（必須に近い）

| ID | 内容 |
|----|------|
| C-1 | ユーザーが **入力用の変動率（%）チャート** に対して、**ポインタで軌道を書く**。テキストによる数値リスト入力は主通路にしない（§2.5）。 |
| C-2 | 描画＋**スケールプリセット**から **各期の変動率 `priceVariations[]`（%）** が決まり、**`calculateDCAResults`** へ渡る。 |
| C-3 | **「シミュレーション実行」** で DCA が走り、**既存の結果表示**（損益率チャート・サマリー・共有）が機能する。 |
| C-4 | **積立回数・横軸・縦軸のいずれかを変更したらストロークは破棄しリセット**する（§8・Q3）。 |

### 2.2 付随（推奨・後続でも可）

- **クリア／やりなおし**：ストロークのみ消して再描画できる。
- **触覚**：スナップ／直線ツールなどは製品次第で検討（初版は任意）。
- アクセシビリティ：キーボードのみでの同等操作はフェーズ分割してよい。

### 2.3 明示的に非ゴールとするもの（当面）

- **実株価のリアルタイム取得**によるチャート自動入力（別機能）。
- 金融商品としての税制・単位変換までの妥当性証明。
- **テキストの数値入力**（キーボードで数字タイプ）は **設定含め全面的に NG**（方針は **2.4** 参照）。

### 2.4 設計決定（Q1〜Q5 — 製品側回答を反映済み）

| # | 決定内容 |
|---|----------|
| **Q1** | チャートの縦軸は **変動率（%）**。**レンジプリセットの具体値は §8.1**（製品提供済み）。 |
| **Q2** | **独立変動モデルのまま**。`calculateDCAResults` は現行どおり **`currentPrice = initialPrice * (1 + variations[i]/100)`** とし、チャート側はそれに適合した **各区間の変動率 %** を手書きサンプルで与える。**実装後の見えを見て再検討可**（フェーズ判定）。 |
| **Q3** | **積立回数・横軸スパン・縦軸レンジのいずれかを変更したら手書きラインは必ずリセット**（§8。画面上の短い説明で許容）。 |
| **Q4** | **モバイルメイン**。手書きの触り心地・追従性を重視。その他レイアウトもモバイル配慮。**元作の反省**：縦スクロールが多いレイアウトにより、描画中に **ページがスクロールしてしまい描線がずれる／描けない**事態がありストレスになった。**本版ではそれを許さない**（次項 **2.5** で UI 要件化）。 |
| **Q5** | **数値入力（テキスト `<input type="number">` など）は一切しない。** 操作は **`select`／スライダー等のみ**。**縦軸・横軸・積立回数の離散候補は §8**。 |

### 2.5 モバイル：描画中スクロール抑止・レイアウト（要件）

設計要件として明示する。

- **入力チャート／キャンパスには**、描画中（`pointerdown`〜`pointerup`/`pointercancel`）にブラウザの **パススクロール（touchmove の既定スクロール）が乗らない**ようにする。**`touch-action`（例：`none` またはドラッガブル領域の局所適用）、必要なら `overscroll-behavior`、`preventDefault`（パッシブでないタッチハンドラ）** 等で、元作級のストレスが再発しないことを優先検証する。
- **スクロールは「描き終わってから」のセクションでもよい**：ただし **描画フェーズ単体viewport** が固定されていて一覧に迷子にならないこと。
- ヒット領域・ボタン寸法・親指での誤タップなど、総合モバイル UI 配慮はワイヤ段階で確認する。

## 3. データモデルとパイプライン（案）

現状との差分のみを明示するため、**共通部分** と **追加部分** に分ける。

### 3.1 共通（現 `lib/dca.ts` と同じ入力契約）

- **入力**：`priceVariationsPercent: number[]`（長さ = 期数）。
- **出力**：`DCARow[]` とサマリー集計ロジック（現状のまま検討可）。

**計算モデル**：**§2.4 Q2** で **独立変動モデルを維持**と決定済み。この前提が変わる場合のみ `dca.ts` を変更し、チャート変換側と同時に合わせる。

### 3.2 追加：チャート層で持つ状態（論理）

概念的には次のような中間状態を用意する。

1. **`variationScale`** — **§8.1「縦軸」**。対称レンジ \([-max,+max]\)（%）。
2. **`horizontalSpan`** — **§8.2「横軸」**。チャート横方向の意味づけ（表示ラベル・目盛。**時間の総幅**として描画領域の目盛りにだけ使う想定）。
3. **`nPeriods`** — **§8.3「積立回数」**。**`sampledVariation[]` の次元**（DCA の配列長）。縦軸・横軸とは独立して指定可能。
4. **`stroke`** — 画面上の複数ポイント。横は正規化 0〜1（左＝開始、右＝`horizontalSpan` の終端）。縦も正規化し、選択 **`variationScale`** にマップして **変動率 %** とする。
5. **`sampledVariation[]`** — ストロークを **横方向に `nPeriods` 本の等間隔ビン**にサンプルした **`number[]`**（長さ `nPeriods`）。**§8 のいずれか変更時は `stroke` 破棄**。

変換関数（名称は実装時に決定）:

```text
stroke + nPeriods + variationScale
  → sampledVariation[]  （長さ nPeriods）
  → calculateDCAResults(sampledVariation[])
```

実装コンテナとしては **`<canvas>` + pointer イベント** か **`SVG` の path編集** か、または **チャートライブラリ（Lightweight Charts / Plotly は描画入力向きではない場合が多い）** で **レイヤーを重ねる**等を比較する（技術検証タスク）。

### 3.3 UI 構成（論理ウィンドウ）

提案する画面上の並び（上から順）。

1. **ヘッダー**（説明：**変動率のイメージをチャートに描いてから**実行する旨）。
2. **入力パネル（§2.4・2.5・§8 準拠）**  
   - UI ラベル **「縦軸」**：**§8.1** を `<select>` 等のみ。変更時 **ストローククリア**。  
   - UI ラベル **「横軸」**：**§8.2** を `<select>` 等のみ。変更時 **ストローククリア**。横は **時間スパンの表示** と手書き領域の目盛対応。**サンプル本数とは別**（サンプル本数は積立回数）。  
   - **積立回数**：**§8.3**（10〜200・10刻み。**スライダーまたはセレクト**のみ、テキスト不可）。変更時 **ストローククリア**。  
   - **手書きチャート**：縦＝選択した縦軸レンジ（%）、横＝選択した横軸スパンの範囲（メイン。**描画中スクロール抑止§2.5**）。  
   - 補助：クリア・アンドゥ任意。変動リストのテキスト編集や数値入力は **置かない**。  
   - （任意）サンプル後の変動だけ折れ線プレビュー（読み取り専用）。
3. **アクション**：「シミュレーション実行」など（十分なヒット領域）。
4. **結果パネル**（`ResultChart`／サマリー／共有）。結果エリアでの縦スクロールは許容だが **描画中は別セクションとの干渉に注意**。

「もう一回」は **結果を畳んでチャートに戻す** とし、**ストロークはクリアまたは維持**は UX で決める（初期案はユーザーが迷わないよう **両方クリア寄りでも可**）。

## 4. 技術検討（実装フェーズでの選択肢概要）

※ 設計フェーズでは **詳細アルゴリズムは確定しない**が、規模感のために列挙する。

| アプローチ | 概要 | メリット | 注意 |
|-----------|------|----------|------|
| Raw Canvas（2d） | 自前で stroke 収集・再描画 | 依存最小、自由度大 | HiDPI・タッチ・アンチエイリアスの手間 |
| SVG path | `<path>` を編集 | スタイルやアクセシビリティの一部有利 | hit-test・平滑化が場合分けされやすい |
| Fabric / Konva 等 | 2Dシーンライブラリ | 開発速度 | 依存・バンドルサイズ増 |
| Chart.jsプラグイン | 既存の Chart.js と統合 | 見た目の統一も可能 | 「描く」機能はプラグインで自作が多い |

**推奨次ステップ**：スパイクで **モバイル実機でのフリードロー + 均等分割サンプル + variationScale プリセット + スクロール抑止** + 既存 DCA に流す、までを PoC で検証する。

## 6. 成果物チェックリスト（「売り」を満たしたか）

実装後、次が **ユーザーに説明できる**状態になれば、元コンテンツのコアとの整合は取れたと見なせる。

- [ ] メイン入力が **変動率チャートへの手書き**で、縦軸は **選択したレンジプリセット**に整合している。

- [ ] 結果画面の説明文が **手書き変動率・独立モデル**と矛盾しない。

## 7. まとめ

- **問題**：現在は **DCA 入力が数値グリッド中心**であり、元の **手書き変動率チャート駆動**と異なる。
- **方向**：**手書き（変動率％）ストローク + スケールプリセット + 均等サンプル**で `priceVariations[]` を生成し、**独立変動モデルの `calculateDCAResults`** に流す。**モバイルで描画中スクロールを抑止**。設定は **数値入力なし**（スライダー／選択のみ）。
- **決定済み**：§2.4（体験・入力方針）、**§8**（縦軸・横軸・積立回数の離散値）。

## 8. 製品提供パラメータ（離散プリセット — 確定）

UI は **すべてプルダウンまたはスライダー／ステップ選択**。テキストで数値を打たせない。  
**既定値**：縦軸 **±100%**、横軸 **5年**、積立回数 **60回**。

### 8.1 縦軸（UI ラベル：`縦軸`）

変動率チャートの **縦方向レンジ**（対称：\(-max \ldots +max\)）。

| 表示（例） | レンジ（%） |
|------------|-------------|
| ±10% | 10 |
| ±30% | 30 |
| ±50% | 50 |
| ±100% | 100 （**既定**） |
| ±500% | 500 |
| ±1000% | 1000 |

### 8.2 横軸（UI ラベル：`横軸`）

手書きエリアおよび目盛りの **時間スパン**の意味づけ。日換算や目盛細分は実装側で決めてよい（下表は参照用。**半年は 182 日相当**など固定してよい）。

| 選択肢 |
|--------|
| 30日 |
| 90日 |
| 半年 |
| 1年 |
| 3年 |
| 5年 （**既定**） |
| 10年 |
| 20年 |
| 30年 |
| 50年 |

※ **積立回数** と独立。例：横軸「5年」・積立回数「60」のとき、画面上は 5年を横幅に均等割りした 60 本のビンにストロークをサンプルする、といった対応になる。

### 8.3 積立回数

- **候補**：10, 20, 30, …, 200（**10 刻み**）
- **既定**：60
- **UI**：スライダー **または** セレクトボックスのみ
